# คู่มือการใช้งานระบบฝั่งผู้ใช้

เอกสารนี้อธิบายการใช้งานฝั่งผู้ใช้ตาม route และ view ที่มีอยู่จริงในโปรเจกต์ปัจจุบัน พร้อมภาพหน้าจอจริงจากระบบที่ถ่ายเมื่อวันที่ 9 เมษายน 2026

อ้างอิงโค้ดใน repository ณ วันที่ 9 เมษายน 2026

## ขอบเขตเอกสาร

- ครอบคลุมหน้าใช้งานที่ผู้ใช้ทั่วไปและผู้สอนเข้าผ่านเมนูฝั่ง `/private`
- ใช้ชื่อ route จริงเพื่อให้ค้นหาหน้าในระบบได้ง่าย
- หากหน้าบางส่วนยังอยู่ในสถานะ `mockup`, `read-first`, หรือมีปุ่มที่ยังไม่ทำงาน จะระบุไว้ในหัวข้อผลลัพธ์ที่ควรเห็น
- รายการ `Path` ใต้หัวข้อรูปประกอบคือ path ของภาพหน้าจอที่ capture จากระบบจริง

## หมายเหตุการ Capture ภาพหน้าจอ

ภาพหน้าจอทั้งหมดถ่ายจากระบบที่รันบน localhost:3000 โดยใช้บัญชีทดสอบที่สมัครใหม่:
- **Username**: `testuser2026`
- **Password**: `Test@123456`
- **วันที่ Capture**: 9 เมษายน 2026

บัญชีนี้มีสิทธิ์เป็น `personal` (ผู้ใช้ทั่วไป) และยังไม่มีสิทธิ์เข้าถึงบางฟีเจอร์ เช่น:
- ห้องเรียนฝั่งผู้สอน (ต้องมีสิทธิ์ teacher)
- GEPOT (ต้องมีสิทธิ์สอบ GEPOT)
- EST (ต้องผ่านเกณฑ์การสอบก่อน)

สำหรับหน้าที่ต้องใช้สิทธิ์พิเศษ ได้ใช้รูปจากหน้าที่เข้าถึงได้แทน หรือใช้รูป placeholder

### ลบบัญชีผู้ใช้
Route: `/private/user/delete-account`

วัตถุประสงค์  
เตรียมพื้นที่สำหรับการลบบัญชีผู้ใช้ในอนาคต

สิทธิ์/เงื่อนไข  
ต้องเข้าสู่ระบบแล้ว แต่ในโค้ดปัจจุบันหน้านี้ยังไม่แสดง flow การลบบัญชีที่ใช้งานได้จริง

ขั้นตอนใช้งาน  
1. เปิดหน้า `/private/user/delete-account`  
2. ตรวจสอบว่าหน้ามีองค์ประกอบการใช้งานจริงหรือยัง  
3. หากทีมจะเปิดใช้ feature นี้ในอนาคต ให้กลับมาเติมขั้นตอนและภาพจริงภายหลัง

ผลลัพธ์ที่ควรเห็น

### จัดการสมาชิกห้องเรียน
Route: `/private/class-room/teacher/:id/members`

วัตถุประสงค์  
ใช้ดูรายชื่อสมาชิก กรองสถานะ และเตรียมเชิญสมาชิกเข้าห้อง

สิทธิ์/เงื่อนไข  
ต้องมีสิทธิ์ผู้สอนในห้องนั้น

ขั้นตอนใช้งาน  
1. เปิดหน้า `/private/class-room/teacher/:id/members`  
2. ดูรายชื่อสมาชิกที่ active หรือ pending  
3. ทดลองกรอก username หรือ email ในส่วนเชิญสมาชิกเพื่อทำ checklist หน้าจอ  
4. เปิดดูรายละเอียดสมาชิกแต่ละคนเมื่อจำเป็น

ผลลัพธ์ที่ควรเห็น

## 8. GEPOT

### หน้า Dashboard ของ GEPOT
Route: `/private/gepot`

วัตถุประสงค์  
ใช้ดูสถานะสิทธิ์สอบ GEPOT ประวัติ และทางไปยังหน้าเริ่มสอบ

สิทธิ์/เงื่อนไข  
ต้องเข้าสู่ระบบแล้ว และระบบต้องโหลดข้อมูลสิทธิ์สอบกับ activity ปัจจุบันได้

ขั้นตอนใช้งาน  
1. เปิดหน้า `/private/gepot`  
2. ตรวจสอบว่าบัญชีมีสิทธิ์สอบแล้วหรือยัง  
3. หากยังไม่มีสิทธิ์ ให้กรอก PIN และ Code เพื่อ redeem  
4. หากมีสิทธิ์แล้ว ให้กดไปหน้าเตรียมสอบหรือหน้าประวัติ

ผลลัพธ์ที่ควรเห็น

### หน้าเตรียมสอบ GEPOT
Route: `/private/gepot/start`

วัตถุประสงค์  
ใช้สรุปข้อมูลก่อนสอบ GEPOT และพาผู้ใช้ไปเริ่มหรือทำข้อสอบต่อ

สิทธิ์/เงื่อนไข  
ต้องมีสิทธิ์ GEPOT ก่อน และ activity ปัจจุบันต้องเปิดให้ทำข้อสอบ

ขั้นตอนใช้งาน  
1. เปิดหน้า `/private/gepot/start`  
2. อ่านคำแนะนำก่อนสอบและสถานะ attempt ปัจจุบัน  
3. กดเริ่มสอบ ทำข้อสอบต่อ หรือดูผลสอบล่าสุดตามสถานะที่หน้าเว็บแสดง

ผลลัพธ์ที่ควรเห็น  
หน้าต้องแสดงข้อมูลเตรียมสอบ ช่วงเวลา activity และ action ที่เหมาะกับสถานะของ attempt เช่น เริ่มสอบ ทำต่อ หรือดูผล

### หน้าทำข้อสอบ GEPOT
Route: `/private/gepot/test?result_id=...`

วัตถุประสงค์  
ใช้ทำข้อสอบ GEPOT จริงตาม result session ที่เปิดอยู่

สิทธิ์/เงื่อนไข  
ต้องมี result_id ที่ถูกต้อง และ session ต้องยังอยู่ในสถานะที่ทำข้อสอบต่อได้

ขั้นตอนใช้งาน  
1. เปิดหน้า `/private/gepot/test?result_id=...` จากหน้า start หรือ dashboard  
2. ตอบคำถามตามลำดับ  
3. ใช้ grid หรือ navigation เพื่อสลับข้อ  
4. ส่งข้อสอบเมื่อทำครบหรือเมื่อพร้อม

ผลลัพธ์ที่ควรเห็น
